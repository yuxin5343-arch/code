# 论文插图引用清单（Figure-to-Claim Mapping）

本文档用于把实验报告中的图表面板，直接映射到论文中的论点与写法模板。
建议在正文引用时统一使用“图X(a)/(b)”编号，不直接依赖 HTML 中的英文标题。

## 使用方式

1. 先打开最新报告（results/report_*.html）。
2. 按本清单给每个子图编号（例如 图6(a)-图6(j)）。
3. 在正文“实验结果分析”中按“现象 -> 原因 -> 结论”三句式引用。

## 图表到论点映射

### 1) Attack Success Rate (Collab vs No-Collab)
- 图意：协同模式与无协同模式的攻击成功率对比。
- 支撑论点：协同闭环可显著降低攻击达成概率。
- 论文句式：
  - 如图X(a)所示，oneshot_collab 的攻击成功率显著低于 no_collab，说明跨域协同决策与执行反馈形成了有效阻断闭环。

### 2) False Negative Rate on Playbook D
- 图意：弱信号场景 D 的漏报率对比。
- 支撑论点：跨域融合可降低弱信号漏报。
- 论文句式：
  - 图X(b)显示，在 D 场景下 no_collab 的漏报率明显高于 oneshot_collab，验证了跨域证据融合对弱信号识别的增益。

### 3) Intent Status Ratios (Collab)
- 图意：一轮协商中 accept/counter/reject 的比例。
- 支撑论点：执行器具有实质自治，而非仅被动执行。
- 论文句式：
  - 从图X(c)可见，counter_proposal 占比非零，表明执行器会基于本地约束主动参与协商。

### 4) Negotiation Game Outcomes
- 图意：counter 总量、采纳率、fallback 率。
- 支撑论点：系统确实发生了“博弈”，且终裁机制可控。
- 论文句式：
  - 图X(d)中 counter_total 与 adopt/fallback 指标共同表明，系统在安全收益与业务约束间执行了可解释的终裁权衡。

### 5) Lateral Spread Block Count
- 图意：横向扩散阻断次数对比。
- 支撑论点：协同机制对跨域扩散具有遏制效果。
- 论文句式：
  - 图X(e)显示协同模式下横向扩散阻断次数更高，反映出跨域联动对攻击链截断能力更强。

### 6) Average Latency (ms)
- 图意：两模式平均时延。
- 支撑论点：协同收益并未引入不可接受时延。
- 论文句式：
  - 图X(f)表明协同模式在获得更高防御效能的同时，时延保持在可接受范围内。

### 7) Low-Confidence Rejected by Manager
- 图意：低置信度阻断被 Manager 拒绝的次数。
- 支撑论点：系统具备“抑制误报扰民”的能力。
- 论文句式：
  - 如图X(g)所示，低置信度阻断请求会被选择性拒绝，说明策略并非一味加码阻断，而是兼顾误报成本。

### 8) Downgrade Adoption Rate (Collab)
- 图意：降级反提案采纳/拒绝占比。
- 支撑论点：协商不是形式化流程，存在真实采纳与拒绝博弈。
- 论文句式：
  - 图X(h)中采纳与拒绝同时存在，说明 Manager 对反提案执行了基于阈值与约束的非刚性终裁。

### 9) Base Risk vs Calibrated Risk (Collab)
- 图意：基础风险分布与校准后风险分布对比。
- 支撑论点：域权重校准机制实际生效，而非仅接入字段。
- 论文句式：
  - 图X(i)显示部分样本由 base_risk 向更高等级迁移，验证了 domain-weight calibration 对风险分级的修正作用。

### 10) Domain-Weight Calibration Metrics
- 图意：平均域权重因子与校准触发率。
- 支撑论点：校准触发具有统计意义且可量化。
- 论文句式：
  - 图X(j)中 risk_calibrated_rate 大于 0，说明域权重校准在批量样本中有稳定触发，而非偶发行为。

## 建议的论文段落骨架

1. 防御效能：先引用图X(a)+图X(e)，证明“能拦截”。
2. 弱信号识别：再引用图X(b)+图X(i)+图X(j)，证明“会校准”。
3. 协商博弈：引用图X(c)+图X(d)+图X(h)，证明“可协商且可解释”。
4. 误报控制：引用图X(g)+图X(f)，证明“不过度阻断且时延可控”。

## 写作注意事项

- 不要只报单点数值，建议同时报告差值或比例变化。
- 结论中区分“机制接入”与“机制生效”（尤其是校准相关指标）。
- 对 no_collab 结果强调其作为基线，不把其解释成缺陷。
